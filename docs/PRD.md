# PRD: Projects Manager

## Статус: Утверждён (MVP)

**Дата:** 2026-04-09

---

## 1. Общее описание

**Projects Manager** — локальное веб-приложение для учёта и отслеживания состояния проектов. Позволяет в любой момент увидеть, на какой стадии находится каждый проект, какие этапы запланированы и какие выполнены. Данные хранятся в markdown-файлах внутри папки каждого проекта.

Приложение в первую очередь является **дашбордом для просмотра** состояния проектов. Основная работа с файлами ведётся через Claude Code. Редактирование через GUI — вспомогательная функция.

## 2. Платформа и архитектура

| Параметр | Значение |
|---|---|
| Этап 1 | Локальное веб-приложение (localhost) |
| Этап 2 (будущее) | Миграция на полноценное веб-приложение |
| Расположение приложения | `G:/PROJECTS_MANAGER/` |
| Расположение проектов | `G:/PROJECTS/` |

---

## 3. Технологический стек

| Компонент | Технология |
|---|---|
| Фреймворк | **Next.js** (full-stack) |
| Язык | **TypeScript** |
| Стилизация | **Tailwind CSS** |
| Иконки | **Lucide Icons** (`lucide-react`) |
| Хранение данных | Markdown-файлы (чтение/запись через Node.js `fs`) |
| Backend | Next.js API Routes (встроенный серверный слой) |

**Обоснование единого стека:**
- Отдельный backend-сервер не нужен — API Routes покрывают все потребности
- Один язык (TypeScript) на всём стеке
- Node.js нативно работает с файловой системой
- При миграции на веб — минимум изменений

```
│   └── ... (код проекта)
├── new_dictionary/
│   ├── docs/
│   │   ├── PRD.md
│   │   ├── PLAN.md
│   │   └── CURRENT_STAGES.md
│   └── ...
└── ...
```

### 4.2. Определение проекта

Папка считается проектом, если содержит подпапку `docs/`. Директории без `docs/` игнорируются.

### 4.3. Файлы проекта

#### PRD.md — описание продукта

Содержит YAML frontmatter с метаданными и полное описание продукта.

```markdown
---
name: "Flat24"
description: "Сервис для поиска и аренды квартир с удобным интерфейсом и фильтрацией"
status: "В работе"
tags: ["веб", "pet-project"]
archived: false
created: "2025-12-01"
last_updated: "2026-04-09"
---

# PRD: Flat24

...полное содержимое PRD...
```

**Поля frontmatter:**
| Поле | Тип | Описание |
|---|---|---|
| `name` | string | Название проекта |
| `description` | string | Краткое описание (отображается на карточке и странице проекта) |
| `status` | string | Текущий статус проекта |
| `tags` | string[] | Теги/категории для фильтрации |
| `archived` | boolean | Флаг архивации |
| `created` | string (date) | Дата создания проекта |
| `last_updated` | string (date) | Дата последнего изменения любого файла проекта |

#### PLAN.md — план работ (крупные этапы)

Составляется на основании PRD. Содержит крупные этапы с кратким описанием. Выполненные этапы отмечаются галочкой.

```markdown
# План работ: Flat24

- [x] **Проектирование** — Составление PRD, определение архитектуры и стека технологий
- [x] **Дизайн** — Разработка макетов интерфейса и UI-kit
- [ ] **Разработка MVP** — Реализация основного функционала: поиск, фильтрация, карточки объектов
- [ ] **Тестирование** — Ручное и автоматизированное тестирование основных сценариев
- [ ] **Деплой и запуск** — Настройка инфраструктуры, CI/CD, первый релиз
```

#### CURRENT_STAGES.md — детализация этапов (подэтапы)

Расшифровка каждого крупного этапа из PLAN.md на мелкие задачи с указанием статуса и даты завершения.

```markdown
# Текущие этапы: Flat24

## Разработка MVP
Статус: В работе

- [x] Инициализация проекта (Next.js, настройка окружения) — завершён 2026-03-15
- [x] Вёрстка главной страницы — завершён 2026-03-22
- [ ] Реализация API поиска объектов — **в работе**
- [ ] Интеграция с базой данных — запланирован
- [ ] Компонент карточки объекта — запланирован
- [ ] Фильтрация и сортировка — запланирован

## Тестирование
Статус: Запланирован

- [ ] Написание unit-тестов для API
- [ ] Тестирование UI-компонентов
- [ ] E2E тестирование основных сценариев
```

## 5. Система статусов

### 5.1. Статусы этапов/подэтапов

| Статус | Описание |
|---|---|
| **Запланирован** | Этап определён, работа не начата |
| **В работе** | Активная работа над этапом |
| **На паузе** | Работа приостановлена |
| **Завершён** | Этап выполнен |
| **Отменён** | Этап отменён, не будет выполняться |

### 5.2. Процент выполнения

- Рассчитывается автоматически для **текущей активной стадии**
- Формула: `(завершённые подэтапы текущей стадии / общее количество подэтапов) × 100%`
- Отменённые подэтапы исключаются из расчёта
- Отображается на карточке проекта на дашборде

## 6. Интерфейс

### 6.1. Дизайн

| Параметр | Значение |
|---|---|
| Стиль | Минималистичный, чистый, интуитивно понятный |
| Ориентир | Главная страница Zoom |
| Темы | Тёмная + светлая с переключением |
| Тёмная тема | Мягкая, тёмно-серые тона (не чисто чёрная) |
| Иконки | Линейные (outline), Lucide Icons |
| Эмодзи | Не используются |

### 6.2. Дашборд (главная страница)

- Список всех проектов в виде карточек
- Строка поиска (по названию проекта и содержимому файлов)
- Фильтрация по тегам, статусу
- Сортировка: по последней активности (по умолчанию), по статусу
- Раздел/фильтр "Архив" для завершённых проектов

**Карточка проекта:**
```
┌─────────────────────────────────┐
│  Flat24                    [веб] │
│  Стадия: Разработка MVP — 50%   │
│  ██████████░░░░░░░░░░           │
│  Статус: В работе               │
└─────────────────────────────────┘
```

### 6.3. Страница проекта (детальный вид)

При клике на карточку — переход на страницу проекта. Принцип: основная информация на одной странице, детали — по клику в модальных окнах.

**Содержимое:**
1. **Краткое описание** — из frontmatter `PRD.md` (вверху страницы)
2. **План выполнения** — все стадии из `PLAN.md` с визуальной отметкой текущего этапа
3. **Детализация текущей стадии** — подэтапы из `CURRENT_STAGES.md` с их статусами
4. **Кнопка "PRD"** — открывает содержимое `PRD.md` в модальном окне

### 6.4. Создание нового проекта

- Кнопка "Новый проект" на дашборде
- При создании генерируется стандартная структура (папка + `docs/` + шаблоны файлов)
- Опция инициализации Git-репозитория (`git init` + начальный коммит)

## 9. Отложенные фичи (Фаза 3+)

| Функция | Причина откладывания |
|---|---|
| Шаблоны проектов | Вернёмся позже по мере накопления типовых паттернов |

| Связи между проектами | Пока нет сценариев использования |

## 10. Приоритеты реализации

1. **Фаза 1 (завершена):** MVP — дашборд, страница проекта, поиск, фильтрация, создание проекта, темы, git-индикация
2. **Фаза 2 (текущая):** хронология активности, заметки, дедлайны, экспорт, аналитика, быстрые действия
3. **Фаза 3 (будущее):** шаблоны, уведомления, связи между проектами, i18n, миграция в веб
