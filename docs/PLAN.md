# План реализации: Projects Manager

## Обзор

План разбит на крупные этапы. Каждый этап — логически завершённый блок, после которого можно оценить результат. Этапы выполняются последовательно.

---

## Фаза 1: MVP (завершена)

- [x] **Этап 1: Проектирование** — Составление PRD, проведение интервью, определение архитектуры и стека технологий
- [x] **Этап 2: Инициализация проекта** — Настройка Next.js, TypeScript, Tailwind CSS, структура папок приложения, базовая конфигурация
- [x] **Этап 3: Backend — API и работа с данными** — API Routes для чтения/записи markdown-файлов, парсинг frontmatter, бизнес-логика (статусы, процент выполнения)
- [x] **Этап 4: Frontend — интерфейс** — Дашборд с карточками проектов, страница проекта, модальные окна, поиск, фильтрация, сортировка, создание проекта
- [x] **Этап 5: Тема и визуальная доработка** — Тёмная/светлая тема, финальная стилизация, адаптивность, polish UI
- [x] **Этап 6: Дополнительный функционал** — Базовое редактирование через UI, архив проектов, инициализация Git при создании проекта

## Фаза 2: Расширение функционала

- [ ] **Этап 7: Хронология активности** — Лента коммитов и изменений проекта в модальном окне (git log, изменения markdown-файлов)
- [ ] **Этап 8: Заметки к проекту** — Быстрые заметки через UI, хранение в docs/notes.md, добавление/просмотр/удаление
- [ ] **Этап 9: Дедлайны и сроки** — Необязательные дедлайны к этапам, индикация просрочки на дашборде, таймлайн на странице проекта
- [ ] **Этап 10: Экспорт отчёта** — Генерация сводного отчёта по проекту в PDF и Markdown
- [ ] **Этап 11: Статистика и аналитика** — Отдельная страница со статистикой по всем проектам (графики, прогресс, активность)
- [x] **Этап 12: Быстрые действия** — Кнопки «Открыть в проводнике» и «Открыть в Cursor» на странице проекта
