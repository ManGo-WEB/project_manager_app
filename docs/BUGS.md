# Баг-трекер: Projects Manager

## Инструкция по ведению BUGS.md

### Назначение

Файл `BUGS.md` — локальный баг-трекер проекта. Хранит историю всех выявленных багов: открытых, исправленных и отклонённых. Позволяет быстро оценить текущее состояние качества проекта и не потерять контекст между сеансами работы.

### Правила заполнения

1. **Каждый баг** оформляется отдельным блоком внутри соответствующего раздела
2. **Формат записи:**
   ```
   ### BUG-{номер}: Краткое описание
   - **Статус:** Открыт | Исправлен | Отклонён
   - **Приоритет:** Критический | Высокий | Средний | Низкий
   - **Обнаружен:** YYYY-MM-DD
   - **Исправлен:** YYYY-MM-DD (если применимо)
   - **Описание:** Подробное описание проблемы
   - **Шаги воспроизведения:** (если применимо)
   - **Решение:** Что было сделано для исправления (если применимо)
   ```
3. **Нумерация** — сквозная, не сбрасывается при переносе в другой раздел
4. **При исправлении** бага — перенести запись из «Открытые» в «Исправленные», добавить дату и описание решения
5. **При отклонении** (не баг, дубликат, won't fix) — перенести в «Отклонённые» с пояснением

### Статусы

| Статус | Описание |
|---|---|
| **Открыт** | Баг обнаружен, ожидает исправления |
| **Исправлен** | Баг устранён, указан коммит/дата |
| **Отклонён** | Не является багом, дубликат или решено не исправлять |

> **Важно:** Этот файл создаётся в `docs/` каждого проекта, управляемого через Projects Manager. Именование — ЗАГЛАВНЫМИ буквами (`BUGS.md`), так как файл напрямую влияет на управление проектом.

## Исправленные

### BUG-3: Бейдж статуса «В работе» переносится на две строки
- **Статус:** Исправлен
- **Приоритет:** Низкий
- **Обнаружен:** 2026-04-11
- **Исправлен:** 2026-04-11
- **Описание:** На дашборде у карточки проекта бейдж статуса «В работе» переносился на две строки из-за нехватки места
- **Решение:** Добавлен `whitespace-nowrap` на компонент StatusBadge (оба варианта — readonly и editable)

### BUG-2: Счётчик заметок не отображается до первого раскрытия раздела
- **Статус:** Исправлен
- **Приоритет:** Низкий
- **Обнаружен:** 2026-04-11
- **Исправлен:** 2026-04-11
- **Описание:** На странице проекта рядом с заголовком «Заметки» не отображалось количество заметок, пока пользователь не раскроет и не схлопнет раздел. Причина — заметки загружались лениво, только при первом клике
- **Решение:** Перенесён вызов `fetchNotes()` в `useEffect` при загрузке страницы, чтобы счётчик был доступен сразу

### BUG-1: Курсор не меняется на pointer при наведении на кнопки
- **Статус:** Исправлен
- **Приоритет:** Низкий
- **Обнаружен:** 2026-04-09
- **Исправлен:** 2026-04-09
- **Описание:** На всех интерактивных элементах (кнопки, ссылки, чекбоксы) курсор оставался стандартной стрелкой вместо pointer
- **Решение:** Добавлены глобальные CSS-правила `cursor: pointer` для всех интерактивных элементов в `globals.css` (коммит `0aa87b4`)
