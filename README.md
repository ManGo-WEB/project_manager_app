# Projects Manager

Локальное веб-приложение для учёта и отслеживания состояния проектов. Позволяет в любой момент увидеть, на какой стадии находится каждый проект, какие этапы запланированы и какие выполнены.

Данные хранятся в markdown-файлах внутри папки каждого проекта — приложение читает их и отображает в удобном интерфейсе.

## Стек технологий

- **Next.js** (full-stack, App Router)
- **TypeScript**
- **Tailwind CSS**
- **Lucide Icons**
- Хранение данных — markdown-файлы с YAML frontmatter

## Установка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/ManGo-WEB/project_manager_app.git
cd project_manager_app

# Установить зависимости
npm install

# Создать файл окружения
echo "PROJECTS_DIR=G:/PROJECTS" > .env.local

# Запустить dev-сервер
npm run dev
```

Приложение будет доступно по адресу `http://localhost:3000`.

## Структура проектов

Приложение работает с проектами, расположенными в папке, указанной в `PROJECTS_DIR`. Каждый проект — это папка с подпапкой `docs/`, содержащей markdown-файлы:

```
G:/PROJECTS/
├── my-project/
│   ├── docs/
│   │   ├── PRD.md              # Описание продукта (с YAML frontmatter)
│   │   ├── PLAN.md             # Крупные этапы работ
│   │   └── CURRENT_STAGES.md   # Детализация этапов
│   └── ...                     # Код проекта
└── another-project/
    └── docs/
        └── ...
```

### PRD.md — метаданные проекта

```yaml
---
name: "My Project"
description: "Краткое описание проекта"
status: "В работе"
tags: ["веб", "pet-project"]
archived: false
created: "2026-01-01"
last_updated: "2026-04-09"
---
```

### PLAN.md — план работ

```markdown
- [x] **Проектирование** — Составление PRD, выбор стека
- [ ] **Разработка MVP** — Реализация основного функционала
- [ ] **Тестирование** — Проверка сценариев
```

### CURRENT_STAGES.md — детализация

```markdown
## Разработка MVP
Статус: В работе

- [x] Инициализация проекта — завершён 2026-03-15
- [ ] Реализация API — **в работе**
- [ ] Вёрстка интерфейса — запланирован
```

## Возможности

### Дашборд
- Список проектов в виде карточек с прогрессом текущей стадии
- Поиск по названию и описанию
- Фильтрация по статусу и тегам
- Сортировка по активности, названию, статусу
- Переключение между активными и архивными проектами
- Git-индикация: незакоммиченные изменения, непушнутые коммиты

### Страница проекта
- Краткое описание и статус (кликабельный для смены)
- План работ с визуальной отметкой текущего этапа
- Детализация этапов с интерактивными чекбоксами
- Редактирование тегов с автоподсказками
- Git-панель: статус рабочей директории, непушнутые коммиты, подключение remote
- Кнопки: PRD (модальное окно), открыть в проводнике, открыть в Cursor

### Создание проекта
- Форма с названием, описанием, тегами (автоподсказки из существующих)
- Автоматическая генерация структуры `docs/` с шаблонами файлов
- Опция инициализации Git-репозитория

### Прочее
- Тёмная и светлая тема
- Глобальные настройки GitHub (username, email)
- Архивация проектов

## Система статусов

| Статус | Описание |
|---|---|
| Запланирован | Работа не начата |
| В работе | Активная разработка |
| На паузе | Работа приостановлена |
| Завершён | Проект выполнен |
| Отменён | Проект отменён |

## Настройки

Страница настроек доступна по иконке шестерёнки в хедере (`/settings`). Содержит:
- **GitHub username** и **email** — используются для `git config` при инициализации репозиториев в новых проектах

Настройки хранятся в файле `settings.json` в корне приложения (не коммитится).

## Структура приложения

```
src/

│   │   ├── projects/       # CRUD проектов, archive, tags, status, git

│   │   └── tags/           # Список всех тегов
│   ├── project/[slug]/     # Страница проекта
│   ├── settings/           # Страница настроек
│   ├── layout.tsx          # Корневой layout
│   └── page.tsx            # Дашборд
├── components/             # React-компоненты
├── lib/                    # Серверная логика (парсеры, работа с файлами)
└── types/                  # TypeScript-типы
```
